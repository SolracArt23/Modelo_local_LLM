API del chatbot con selección de archivo

La API permite consultar el modelo indicando explícitamente el archivo PDF que se usará como contexto y el prompt que se enviará al modelo.

Para usar esta integración ya no necesitas iniciar sesión en la web: debes enviar el token del usuario propietario en cada petición.

Autenticación por token

Puedes enviar el token de cualquiera de estas formas:

  • Authorization: Bearer TU_TOKEN
  • X-API-Token: TU_TOKEN
  • En el cuerpo JSON como token o widget_token
  • En multipart/form-data como token al subir archivos

Ese token lo obtienes desde el panel principal del sistema, en la sección de créditos y widget.

1. Subir un archivo

Primero sube el PDF a la carpeta de trabajo del chatbot usando el token del usuario.

POST /upload

Envía el archivo en multipart/form-data con la clave file y agrega el token.

curl -X POST http://127.0.0.1:5000/upload \
    -H "X-API-Token: TU_TOKEN" \
    -F "file=@manual_usuario.pdf" \
    -F "description=Manual del sistema" \
    -F "casos_de_uso=Consultas operativas"

2. Ver archivos disponibles

GET /api/files
curl -X GET http://127.0.0.1:5000/api/files \
    -H "Authorization: Bearer TU_TOKEN"

3. Consultar el chatbot

POST /api/chat
Content-Type: application/json
Authorization: Bearer TU_TOKEN

{
    "token": "TU_TOKEN",
  "archivo": "manual_usuario.pdf",
  "prompt": "Resume los puntos principales del documento"
}

Respuesta esperada

{
  "archivo": "manual_usuario.pdf",
  "modelo": "llama3.1:latest",
  "prompt": "Resume los puntos principales del documento",
  "respuesta": "..."
}

Ejemplo con cURL

curl -X POST http://127.0.0.1:5000/api/chat \
    -H "Authorization: Bearer TU_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
        "token": "TU_TOKEN",
    "archivo": "manual_usuario.pdf",
    "prompt": "Que informacion relevante contiene este archivo?"
  }'
